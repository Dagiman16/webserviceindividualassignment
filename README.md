# ShapeTool SOAP Web Service

Woldia University – Software Engineering 5th Year  
Web Service Assignment 1

## Overview

ShapeTool is a contract-first SOAP web service built with Spring Boot and Spring-WS. It calculates the area of five geometric shapes: circle, square, rectangle, parallelogram, and triangle.

## Service Operations

| Operation | Parameters | Formula |
|---|---|---|
| circleArea | radius | π × r² |
| squareArea | side | s² |
| rectangleArea | length, width | l × w |
| parallelogramArea | base, height | b × h |
| triangleArea | base, height | 0.5 × b × h |

## Endpoint

- WSDL: `http://localhost:8080/ws/calculator.wsdl`
- Endpoint URL: `http://localhost:8080/ws`

## XSD Design

The schema is defined in `src/main/resources/schemas/calculator.xsd`.

- Namespace: `http://example.com/shapetool`
- All input values use a custom `positiveDouble` restricted type:
  ```xml
  <xs:simpleType name="positiveDouble">
      <xs:restriction base="xs:double">
          <xs:minExclusive value="0"/>
      </xs:restriction>
  </xs:simpleType>
  ```
- This restriction prevents zero and negative values at the schema level.
- Each operation has its own request and response element (document/literal style).

## WSDL

The WSDL is auto-generated by Spring-WS from the XSD at runtime. It defines all 5 operations, their input/output messages, binding (SOAP 1.1, document/literal), and the service endpoint.

## Validation Strategy

Two layers of validation:

1. XSD restriction — `positiveDouble` type rejects zero and negative values at the schema level before the request reaches the endpoint.
2. Java validation — `ShapeEndpoint` checks each parameter with `validatePositive()` and throws `IllegalArgumentException` for any invalid value.

## SOAP Fault Handling

`SoapFaultConfig` registers a `SoapFaultMappingExceptionResolver` that maps:
- `IllegalArgumentException` → `SOAP-ENV:Client` fault (bad input from caller)
- Any other exception → `SOAP-ENV:Server` fault

This ensures all errors return a proper SOAP Fault message instead of an HTTP 500.

## How to Run

```bash
./mvnw spring-boot:run
```

The service starts on port 8080.

## Testing with SoapUI or Postman

1. Import the WSDL from `http://localhost:8080/ws/calculator.wsdl`
2. Use the sample request/response XML files in the `samples/` folder
3. Send requests to `http://localhost:8080/ws`
